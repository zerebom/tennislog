## 不合格項目の詳細と推奨修正

### 致命的（Critical） -- 図として機能していない

#### C1. 矢印が表示されていない（#11, #15-19, #33）

**問題**: PNG上で矢印（エッジ）が一切レンダリングされていない。ノードだけが浮いて表示されている状態で、フロー図としての意味をなさない。

**原因の可能性**: edgeのparent属性がすべて"1"（ルート）になっている。レーンをまたぐedgeはparent="1"で正しいケースもあるが、レンダリングエンジンの挙動次第で非表示になる可能性がある。または、PNGへの変換時にedgeがレンダリングされなかった可能性。

**推奨修正**:
1. draw.ioのデスクトップアプリまたはWeb版で.drawioファイルを開き、矢印が表示されるか確認する
2. 表示されない場合、edgeを再描画するか、edgeのスタイルに明示的なルーティング情報を追加する
3. PNGへの再変換を実施する

#### C2. f_scan と f_tap が完全に重なっている（#9）

**問題**: 「QRスキャン」(f_scan)と「リンクタップ」(f_tap)が相手レーン内のまったく同じ座標(x=590, y=55)に配置されている。PNG上では「リンクタップ」のみが見え、「QRスキャン」が完全に隠れている。

**推奨修正**: f_scanとf_tapは分岐の並列パスなので、Y座標をずらすか、X座標を変えて配置する。ただしレーン高さ(160px, ヘッダー除くと120px)に対して2つのノードを縦に並べるのは狭いため、レーンの高さ拡大も検討。

例:
```xml
<!-- f_scan を上側に配置 -->
<mxGeometry x="590" y="45" width="130" height="50" as="geometry" />
<!-- f_tap を下側に配置 -->
<mxGeometry x="590" y="100" width="130" height="50" as="geometry" />
```
ただし、レーンの高さを200px程度に拡大することを推奨。

### 重要（Major） -- レイアウト品質の問題

#### M1. フォントサイズが基準未満（#4）

**問題**: すべてのノードのfontSizeが14で、チェックリスト基準の18未満。

**推奨修正**: すべてのノードおよびedgeラベルのfontSizeを18に変更。ノードのサイズもそれに合わせて拡大する必要がある（幅+20〜30px、高さ+10〜20px程度）。

#### M2. 水平ギャップ不足（#20）

**問題**:
- lane1: f_start→f_method = 70px（基準120px未満）
- lane1: f_method→f_qr = 80px（基準120px未満）
- lane3: f_scan/f_tap→f_notify = 80px（基準120px未満）

**推奨修正**: 図全体の幅を広げるか（mxGraphModel dx/pageWidth拡大）、ノード間隔を調整する。lane1の左側3ノードは特に詰まっており、各ノード間に最低120pxの水平ギャップを確保する。

#### M3. 垂直マージン不足（#21）

**問題**: 多数のノードがレーンのコンテンツ領域の上端/下端から40px未満の位置にある。
- f_method: top_margin = 5px
- f_approve: top_margin = 0px（ヘッダーと接触）
- f_start: top_margin = 15px
- f_qr, f_done: top_margin = 10px

**推奨修正**: レーンの高さを200〜220pxに拡大し、ノードをレーンの中央付近に再配置する。startSizeが40pxなので、コンテンツ領域は160〜180pxとなり、ノードの垂直中央配置で40px以上のマージンを確保可能。

#### M4. クロスレーン矢印が垂直でない（#22）

**問題**: レーンをまたぐ矢印のsource/targetのX中心座標差が大きく、斜めに横断する形になる。
- e3: f_method→f_link = X差200px（FAIL、基準50px以内）
- e4: f_qr→f_scan = X差190px（FAIL）
- e5: f_link→f_tap = X差190px（FAIL）
- e9: f_approve→f_mutual = X差165px（FAIL）

**推奨修正**: クロスレーンの接続先ノードを、接続元と同じX位置に揃える。具体的には:
- f_link(アプリ)のX座標をf_method(自分)の直下に移動（x=210付近）
- f_scan, f_tapのX座標をf_qr, f_linkの直下に移動（x=400付近）
- あるいは、図全体のレイアウトを再設計し、各ステップが縦に揃うようにする

#### M5. f_approve と f_reject の重なり/接触（#10）

**問題**: f_approve(x=970, y=40, h=90)の下端がy=130、f_reject(x=970, y=100)の上端がy=100。f_approveの範囲(y=40〜130)とf_reject(y=100〜145)が重なっている。

**推奨修正**: f_rejectの位置をf_approveの下に十分な間隔をもって配置する。レーン高さの拡大が前提。
