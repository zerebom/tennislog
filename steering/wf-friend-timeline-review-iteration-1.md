## 不合格項目の詳細と推奨修正

### 【致命的】問題1: ノードの完全重複 (チェック #9, #20)

**現象**: `t_display`（タイムライン表示）と `t_reflect`（記録をTLに反映）が、アプリレーン内でまったく同じ座標 `(x=490, y=55, w=150, h=70)` に配置されている。PNGでは片方のノードが完全に隠れている。

**影響**: プロセスの一部が見えなくなり、フロー全体の理解が不可能。

**推奨修正**:
- `t_reflect` の x 座標を `490` から別の位置に移動する。フロー順序を考慮すると、`t_friend_record` → `t_reflect` → `t_display` の流れで配置するのが自然。
- 案: `t_reflect` を x=490 に維持し、`t_display` を x=700 に移動。またはフロー全体のレイアウトを再設計。

```xml
<!-- t_reflect: 現状維持 x=490 -->
<mxCell id="t_reflect" ... >
  <mxGeometry x="490" y="55" width="150" height="70" as="geometry" />
</mxCell>

<!-- t_display: x を 700 に変更 -->
<mxCell id="t_display" ... >
  <mxGeometry x="700" y="55" width="150" height="70" as="geometry" />
</mxCell>
```

### 【致命的】問題2: PNG上に矢印が描画されていない (チェック #11, #15-19, #33)

**現象**: drawio XMLには8本の矢印（e1〜e8）が定義されているが、PNGレンダリングでは矢印が一切表示されていない。ノードだけが孤立して配置されている。

**影響**: スイムレーン図として最も重要なフロー（プロセスの流れ）が視覚的に伝わらない。

**推奨修正**:
- 矢印の `edge="1"` 属性が `parent="1"` に対して設定されているが、ソース/ターゲットが異なるレーンにある場合のルーティングに問題がある可能性。PNGの再変換（`convert_to_png.sh`）を実施して矢印が描画されるか確認。
- それでも表示されない場合、エッジ定義を各レーンの `parent` ではなくルート `parent="1"` のままで正しいか確認し、`strokeWidth=2` などスタイルを追加して可視化を確保。

### 【重要】問題3: フォントサイズ不足 (チェック #4)

**現象**: タイトル（fontSize=22）以外のすべてのノードが `fontSize=14` を使用している。チェックリスト基準は `fontSize=18` 以上。

**推奨修正**: すべてのノード・ラベルの `fontSize` を `18` に変更。ノードのサイズもテキストが収まるよう調整が必要。

```
fontSize=14 → fontSize=18
```

### 【重要】問題4: クロスレーン矢印の垂直整列不足 (チェック #22)

**現象**: レーンをまたぐフローで、ソースとターゲットのX中心座標の差が大きい。

| 矢印 | ソースX中心 | ターゲットX中心 | 差 | 判定 |
|------|-----------|---------------|-----|------|
| e1: t_start → t_fetch | 210 | 395 | 185px | FAIL (>50px) |
| e3: t_friend_record → t_reflect | 385 | 685 | 300px | FAIL (>50px) |
| e5: t_display → t_like | 685 | 875 | 190px | FAIL (>50px) |
| e6: t_like → t_notify_send | 875 | 1065 | 190px | FAIL (>50px) |
| e7: t_notify_send → t_notify_recv | 1065 | 1060 | 5px | PASS |

**推奨修正**: クロスレーン矢印が垂直に近くなるよう、ソースとターゲットのX位置を揃える。特に e1, e3, e5, e6 は大幅な斜め移動になっている。ノード配置の全面的な見直しが必要。

### 【軽微】問題5: t_like と t_notify_send の水平ギャップ不足 (チェック #20)

**現象**: `t_like`（lane1内 x=690, w=130, 右端=820）と `t_notify_send`（lane2内 x=880）は異なるレーンのため直接比較は参考値だが、同じ水平ラインで見ると間隔は60px程度。ただしこれはクロスレーンなので致命的ではない。

## フロー設計への提案

現在のXML定義のフローを整理すると:

```
[自分] フレンドタブ選択 (t_start)
  ↓ e1
[アプリ] タイムラインデータ取得 (t_fetch)
  ↓ e2
[アプリ] タイムライン表示 (t_display) ← 記録をTLに反映 (t_reflect) からも入力 (e4)
  ↓ e5
[自分] いいね/コメント (t_like)
  ↓ e6
[アプリ] 友達に通知送信 (t_notify_send)
  ↓ e7
[友達] 通知受信 (t_notify_recv)
  ↓ e8
[友達] モチベUP (t_done)

並行して:
[友達] 試合/練習を記録 (t_friend_record) → e3 → [アプリ] 記録をTLに反映 (t_reflect) → e4 → t_display
```

このフローでは `t_display` に3系統の入力（e2, e4）があり合流点になっている。レイアウト上、左から右の時系列に沿うよう以下の配置を推奨:

```
時間軸 →
列1        列2            列3              列4              列5            列6
t_start → t_fetch      → t_reflect      → t_display      → t_like      → t_notify_send
           t_friend_record ↗ (e3)                                         ↓ e7
                                                                        t_notify_recv → t_done
```
