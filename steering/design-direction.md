# TennisLog デザインディレクション

モックのデザインレビューと、改善の方向性をまとめる。

---

## 0. v2での改善点（対応済み）

v1→v2で以下が修正された。design-direction v1で指摘した課題の大半が解消されている。

- オンボーディングフロー（Welcome → Profile → 初回記録誘導）
- タイプ選択画面（練習/シングルス/ダブルス）
- 😤😐😊 が全記録フォームに入った（練習・シングルス・ダブルス共通）
- 練習記録フォーム（タグ選択 + 調子 + メモ）
- ダブルス記録フォーム（パートナー選択）
- 完了画面にインサイトカード（TODAY'S INSIGHT、ランダム表示）
- ホームに「調子の推移」折れ線グラフ（SVG）
- タイムラインに練習カードが統合（アンバー色で差別化）
- カレンダーに練習日表示 + 凡例追加
- ボトムタブ・ヘッダーを「仲間」に統一
- H2H詳細の各試合に😤😐😊表示
- パートナーカード（山本さん：「パートナー」バッジ + ペア成績）

**評価**: 機能面の抜け漏れはほぼ解消。design.mdの仕様がモックに正しく反映された。残りの課題は「見た目の質」と「design.mdとの細かい乖離」。

## 1. v2の残課題

### 1-1. ビジュアル：「どこかで見たアプリ」問題（未対応）

v2でも色・フォント・テクスチャは変わっていない。

- **カラーパレットがTailwindデフォルトのまま**: `#16A34A`（green-600）、`#EF4444`（red-500）
- **テクスチャ・素材感なし**: フラットな白カード + 薄グレー背景
- **フォントに表情がない**: Inter一辺倒。スコア「6-4」が本文と同じ書体

### 1-2. design.mdとの乖離（新規指摘）

#### オンボーディングに「レベル感」が追加されている

design.mdのUserProfileには `experience`（テニス歴）しかない。v2のオンボーディングStep 2に「レベル感（初級/中級/中上級/上級）」が追加されているが、データモデルに対応フィールドがない。

**判断**: 入れるならdesign.mdのデータモデルを更新する。入れないならモックから削除する。個人的にはオンボーディングの入力を増やすべきではない（離脱リスク）。テニス歴だけで十分。

#### 仲間カードにレベル・プレースタイルがまだ表示されている

design.mdでは「名前だけで登録可能」「レベルは2回以上対戦後にアプリが提案」。v2の仲間カードには依然として「中上級」「ストローカー」等がプリセットで表示されている。

**あるべき姿**: 一覧カードには名前 + 勝敗 + 最終日のみ。レベル・プレースタイルは詳細ドロワー内に留める。

#### 練習カードに「時間」が表示されている

v2の練習カードに「1.5h」「0.5h」が表示されているが、design.mdのPracticeインターフェースに`duration`フィールドがない。入れるなら追加、入れないなら削除。

**判断**: 時間は記録モチベーションに寄与しにくい（何時間やったかより何をやったかが重要）。P0では不要。

#### 試合カードに「場所」がまだ表示されている

v1から引き続き「有明テニスの森」「世田谷公園」等が表示。データモデルにlocationフィールドがない。

**判断**: 場所は振り返りの文脈として有用（「あのコートで勝てた」）。入れるならデータモデルに`location?: string`を追加。ただしP0の入力項目を増やすのは慎重に。P1以降が妥当。

#### シングルス記録フォームにP1のスキルタグが混在

シングルスフォームの「メモ」セクションに「サーブ好調」「リターンミス」「ネットプレー◎」等のタグチップがある。これはdesign.mdでP1に定義されている「スキルタグ」であり、P0のMVPには入れない方針。

練習フォームの「取り組みタグ」（サーブ/ストローク/ボレー等）とは別物。混同しやすいのでP0では削除すべき。

### 1-3. 空状態が一切ない（未対応）

v2でも全画面が「データが溜まった状態」のみ。

MVP初期ユーザーは全員「記録0件」からスタートする。**一番多くのユーザーが最初に見る画面**が設計されていない。

考えるべき空状態：
- **ホーム画面**: 記録0件 → 月間サマリー「0試合/0%/0連勝」は辛い
- **調子の推移グラフ**: データ0-2件 → グラフが成立しない
- **仲間一覧**: 仲間0人
- **H2Hドロワー**: 特定の相手との試合が1件だけ

### 1-4. ホームの情報順序（部分対応）

v2で調子の推移グラフが追加されたが、画面の順序は：

```
[月間サマリー（数値）] ← まだ数値が先
[調子の推移グラフ]    ← 追加されたが2番目
[タイムライン]
```

design-direction v1の提案は「感情ファースト」。グラフが入ったのは良いが、**順序を入れ替えるだけで印象が変わる**：

```
[調子の推移 + 一言インサイト]  ← 感情が先
[月間サマリー]
[タイムライン]
```

### 1-5. 細かいUI品質

- **text-[10px]多用**: カレンダーの曜日、凡例、バッジ等。実機では読みにくい。最小11px推奨
- **ボトムナビのタブ構造**: design.mdでは「ホーム / 仲間 / 統計」の3タブ + FAB。v2は「ホーム / FAB（記録） / 仲間」で統計タブがない（v0.2以降なので現時点ではOKだが、タブ枠をFABが食っている構造は将来の拡張時に問題になる）
- **バッチ記録（まとめ記録）**: タイプ選択画面にない。草トーユースケースの対応漏れ

## 2. デザインの方向性（v2ベースで更新）

### 2-1. カラーパレットの調整（最小工数で最大効果）

Tailwind configを変えるだけ。コード変更はほぼゼロ。

```
現状                        → 提案
primary: '#16A34A'          → '#1B7A4A'（深みのある芝コートグリーン）
lose: '#EF4444'             → '#D4483B'（テラコッタ寄りの赤）
surface: '#F8FAFC'          → '#F7F5F0'（わずかにベージュ）
text-primary: '#1E293B'     → '#1A2B1F'（グリーンが微量入った深い色）
```

**負けの赤をテラコッタに寄せる理由**: 真っ赤(#EF4444)は「失敗・エラー」の連想が強い。負けは「成長の材料」であり、テニスのクレーコートを連想させる落ち着いた赤の方がプロダクトの世界観に合う。

### 2-2. ホーム画面の順序入れ替え

```
[ヘッダー: TennisLog + アバター]
[調子の推移グラフ + 一言コメント]  ← 最初に目に入る
[月間サマリー + カレンダー]
[直近のアクティビティ]
```

調子の推移グラフの下にある「😊 が3試合連続！好調持続中」のテキストが非常に良い。これを月間サマリーカードの中に統合して「数値 + 一言」にするとさらに良い：

```
3月のサマリー
8試合 / 62% / 5回練習 / 3連勝中🔥
"😊が3試合連続！先月を上回るペースです"
[カレンダー]
```

### 2-3. 仲間カードの簡素化

```
現状:
[田] 田中さん [中上級]
     ストローカー
     3勝 2敗 | 60% | 最終 3/1

提案:
[田] 田中さん
     3勝 2敗 (60%) | 最終 3/1
```

レベルバッジとプレースタイルを一覧から外す。詳細ドロワーに集約。

### 2-4. 空状態の設計

各画面に空状態を追加する。

**ホーム（記録0件）**:
```
🎾
まだ記録がありません

テニスの後にサクッと記録して
上達を見える化しましょう

[最初の記録をする]
```

**調子の推移（3件未満）**:
```
📊 あと○回記録すると
調子の推移グラフが表示されます
```

**仲間（0人）**:
```
👥
試合を記録するとき、
相手やパートナーの名前を入れると
ここに自動で追加されます

[試合を記録する]
```

### 2-5. スコア入力のドラムロール化（P2）

`<select>` → iOS picker風のドラムロール。実装コストは中程度だが、テニスのスコア入力としての「らしさ」が大幅に上がる。ライブラリ: `react-mobile-picker`。

### 2-6. 数字フォントの導入（P3）

スコア・勝率・連勝数に DM Mono または Space Mono を適用。スコアボードの空気を出す。Google Fontsから読み込むだけで実現可能。

## 3. 優先順位（v2ベースで更新）

v2で旧P0は全て対応済み。新しい優先度：

| 優先度 | 改善項目 | 理由 | 工数 |
|-------|---------|------|------|
| **P0** | オンボーディングから「レベル感」を削除 | design.mdに対応フィールドがない。入力増は離脱リスク | 極小 |
| **P0** | シングルスフォームからスキルタグを削除 | P1機能がP0に混入している | 極小 |
| **P0** | 練習カードから時間表示を削除（or データモデル更新） | design.mdとの不整合 | 極小 |
| **P1** | カラーパレットの調整 | Tailwind config数行で世界観が変わる | 小 |
| **P1** | ホーム画面の順序入れ替え（調子グラフを先に） | 「感情ファースト」の体現 | 小 |
| **P1** | 仲間カードの簡素化（レベル・スタイル非表示） | design.mdとの整合 | 小 |
| **P1** | 空状態のデザイン追加 | MVP初期ユーザー全員が見る画面 | 中 |
| **P2** | スコア入力のドラムロール化 | テニスらしい入力体験 | 中 |
| **P2** | タイプ選択にバッチ記録を追加 | 草トーユースケース対応 | 中 |
| **P2** | text-[10px]の11px以上への引き上げ | 実機での可読性 | 小 |
| **P3** | 数字フォント（DM Mono等）導入 | 雰囲気の仕上げ | 小 |
| **P3** | 完了画面のアニメーション強化 | Variable Rewardの演出磨き | 中 |

## 4. やらないこと

- **ダークモード**: MVPでは不要
- **カスタムイラスト/ロゴ**: テキスト+絵文字で十分
- **複雑なアニメーション**: Lottie等は不要。CSSトランジションで十分
- **ブランドガイドライン**: MVP段階では早い

## まとめ

v2でdesign.mdの仕様は概ね正しくモックに反映された。**機能の骨格は固まった**。

残りの改善は2軸：

1. **design.mdとの細かい不整合を潰す**（P0: レベル感削除、スキルタグ削除、時間削除）
2. **ビジュアルの質を上げる**（P1: カラー調整、順序入れ替え、空状態追加）

v2は「正しく動くプロトタイプ」。ここからP0の不整合を修正し、P1のカラー+順序+空状態を入れれば「触りたくなるプロトタイプ」になる。
