# implement-feature

タスク指示を読み、React コンポーネント/ページ/ストアを実装する手順。

## 手順

1. **タスク指示を確認**: 修正対象のファイル、修正内容、期待される動作を把握する
2. **対象ファイルを読む**: 修正対象のファイルの現在の実装を読む
3. **関連ファイルを確認**: 型定義 (`src/types.ts`)、関連ストア (`src/stores/`)、関連コンポーネント (`src/components/`) を確認
4. **修正を実装する**: タスク指示に従って実装する
5. **型チェック実行**: `npx tsc --noEmit` でエラーがないことを確認する
6. **スコープ確認**: タスク指示に書かれていない変更をしていないことを確認する

## ルール

- 既存のコンポーネントインターフェース（Props型）を不必要に変えない
- タスク指示に書いていない機能を追加しない
- 型チェックが通らなければ自分で修正する（最大3回リトライ）
- 3回リトライしても通らない場合は、エラー内容をレポートして停止する
- パスエイリアス `@/` を使う
- shadcn/ui + Tailwind CSS 4 でスタイリング
- `any` 型を使わない
